# Configuration

## Flow config (`config.yml`)

Each flow owns a `config.yml` that declares its inputs, outputs, and the output registry.

### Schema

```yaml
# Inputs
input_dir: "raw"
input_registry: "raw/registry.yml"

# Secondary inputs (convention: input_dir_<name> + input_registry_<name>)
input_dir_brainstate: "derivatives"
input_registry_brainstate: "derivatives/brainstate/registry.yml"

# Outputs
output_dir: "derivatives/preprocess"
output_registry: "derivatives/preprocess/registry.yml"

# Output registry (the data contract)
registry:
  raw_zarr:
    base_input: "ieeg"
    bids:
      root: "raw_zarr"
      datatype: "ieeg"
    members:
      zarr: { suffix: "ieeg", extension: ".zarr" }

# Workflow-engine-specific fields (passed through, not validated)
pybids_inputs:
  ieeg:
    filters: { suffix: "ieeg" }
    wildcards: [subject, session]

# Stage aliases (optional)
stage_aliases:
  labels: badlabel
```

### Field reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `input_dir` | string | yes | Root directory for input data |
| `input_registry` | string | no | Path to input registry YAML |
| `output_dir` | string | yes | Root directory for outputs |
| `output_registry` | string | no | Path to output registry file |
| `registry` | mapping | yes | Output registry (data contract) |

### Registry group fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `base_input` | string | no | Which input key drives entity expansion |
| `bids` | mapping | no | BIDS directory structure (`root`, `datatype`) |
| `members` | mapping | yes | Named output products (`suffix`, `extension`) |

### Extra inputs

The convention `input_dir_<name>` + `input_registry_<name>` declares secondary input sources from other pipelines' outputs. Discovered via `FlowConfig.extra_inputs()`.

## Pipeline registry (`.pipeio/registry.yml`)

Generated by `pipeio registry scan`. Maps flows to filesystem paths.

```yaml
flows:
  preprocess/ieeg:
    name: ieeg
    pipe: preprocess
    code_path: code/pipelines/preprocess/ieeg
    config_path: code/pipelines/preprocess/ieeg/config.yml
    doc_path: null
    mods: {}
```
